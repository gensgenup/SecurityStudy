# Repository Policy

Updated: 2026-02-15

## Classification

- Status: `incubating`
- Visibility: `public`
- Purpose: security study notes and reproducible practice code.

## Publishing Rules

- Never publish exploit code tied to real targets.
- Remove credentials, private logs, and sensitive host identifiers.

## Branch Governance

- Default branch is `main`.
- Branch protection must enforce pull requests and one approval.
- Force-push and branch deletion on `main` are disabled.

## Maintenance

- Run a monthly review for stale notes and broken links.
- Keep legal and ethical boundaries documented in README.
